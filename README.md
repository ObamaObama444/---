# Open Room PHP

Форум без регистрации на `PHP FastCGI`: один общий чат, обмен файлами, drag-and-drop, вставка картинок из буфера и хранение истории на диске. Пользователь открывает сайт и сразу попадает в чат как гость.

## Что умеет

- вход без логина и пароля
- общий чат с polling раз в 2 секунды
- отправка файлов через скрепку, drag-and-drop и вставку изображений
- встроенный показ картинок в ленте
- хранение истории в `storage/messages.json`
- хранение файлов в `storage/uploads/`
- запуск на обычном `PHP FastCGI` без `Node.js`, `pm2` и `npm`

## Структура

- `index.php` — публичная страница с маскировкой под docs-сайт
- `assets/` — клиентский JS и CSS
- `api/` — PHP API для сессии, сообщений, файлов и healthcheck
- `storage/` — история и файлы

## Деплой на PHP FastCGI

1. Укажи корнем сайта директорию этого репозитория
2. Убедись, что у домена включен `PHP FastCGI`
3. Проверь, что PHP может писать в `storage/` и `storage/uploads/`
4. Открой сайт

Никаких `npm install`, `node server.js` и `pm2` не нужно.

## Развертывание из Shell-клиента

Если сайт уже создан в панели и его корневая директория указывает на папку домена:

```bash
find . -mindepth 1 -maxdepth 1 ! -name '.well-known' -exec rm -rf {} +
git clone https://github.com/ObamaObama444/--- .
chmod -R 775 storage
```

Важно: первая команда очищает текущую директорию сайта, кроме `.well-known`.

## Проверка после деплоя

Открой:

```text
https://твой-домен/api/health.php
```

Если все ок, увидишь JSON с `ok: true`.

## Хранилище данных

- сообщения и метаданные файлов пишутся в `storage/messages.json`
- сами файлы пишутся в `storage/uploads/`
- папка `storage/` закрыта от прямого HTTP-доступа через `.htaccess`

## Ограничения текущей версии

- чат обновляется polling-ом, а не WebSocket
- лимит одного файла: `25 МБ`
- хранится до `250` последних сообщений
- чат общий, без комнат и без приватных переписок
