# Open Room

Realtime-форум без регистрации: один общий чат, обмен файлами, drag-and-drop, вставка изображений из буфера обмена и публичный запуск через туннель. Пользователь просто открывает сайт и сразу попадает в чат как гость.

## Что умеет

- вход без логина и пароля
- realtime-чат на `Socket.IO`
- отправка файлов через скрепку, drag-and-drop и вставку изображений из буфера обмена
- встроенный показ картинок в ленте
- хранение истории в `data/messages.json`
- хранение файлов в `uploads/`
- публичный запуск одной командой

## Быстрый старт

1. Установить зависимости:

   ```bash
   npm install
   ```

2. Запустить локально:

   ```bash
   npm start
   ```

3. Открыть форум:

   ```text
   http://localhost:8000
   ```

## Публичный запуск

Если нужен внешний URL, который можно открыть с любого устройства, выполни:

```bash
npm run public
```

Скрипт:

- сам запускает форум на `http://localhost:8000`
- поднимает публичный HTTPS-туннель через `localhost.run`
- печатает внешний URL прямо в терминал

После этого любой человек сможет зайти по выданной ссылке, пока открыт терминал и работает интернет на устройстве.

### Важно про публичную ссылку

- по умолчанию это будет публичный домен туннеля, а не твой собственный домен
- ссылка может измениться после перезапуска
- для постоянного красивого домена вроде `chat.example.com` позже лучше подключить свой домен и отдельный туннель или reverse proxy

## Сброс форума

Очистить историю сообщений и все загруженные файлы:

```bash
npm run reset
```

## Запуск на старом ноуте

Самый простой рабочий вариант:

1. Поставить Node.js LTS и Git
2. Клонировать репозиторий
3. Выполнить `npm install`
4. Для локального режима использовать `npm start`
5. Для внешнего доступа использовать `npm run public`

Если хочешь, чтобы форум жил после перезагрузки, можно использовать `pm2`:

```bash
npm install -g pm2
pm2 start ecosystem.config.js
pm2 save
```

После этого форум будет локально работать на `8000`, а публичный URL можно поднимать отдельно командой `npm run public`.

## Хранилище данных

- сообщения сохраняются в `data/messages.json`
- файлы сохраняются в `uploads/`
- данные не отправляются в GitHub автоматически
- если ноут выключен, спит или интернет пропал, форум и файлы будут недоступны

## Ограничения текущей версии

- лимит на один файл: 25 МБ
- хранится до 250 последних сообщений
- без своей DNS-настройки публичная ссылка не будет постоянным красивым доменом
