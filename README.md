# Open Room

Форум без регистрации: один общий чат, обмен файлами, drag-and-drop, вставка изображений из буфера обмена и хранение истории на сервере. Пользователь открывает сайт и сразу попадает в чат как гость.

## Что умеет

- вход без логина и пароля
- realtime-чат на `Socket.IO`
- отправка файлов через скрепку, drag-and-drop и вставку изображений из буфера
- показ картинок прямо в ленте
- хранение истории в `data/messages.json`
- хранение файлов в `uploads/`
- запуск локально, через `pm2` или через `Node.js`-сайт в `ispmanager`

## Быстрый старт

```bash
npm install
npm start
```

После запуска форум будет доступен на:

```text
http://localhost:8000
```

## Сброс форума

Очистить историю сообщений и загруженные файлы:

```bash
npm run reset
```

## Развертывание на REG.RU через ispmanager

Проект уже подготовлен под обычный домен без туннелей. Сервер умеет принимать:

- `PORT` и `INSTANCE_HOST` для режима `Node.js -> Port`
- `SOCKET` для режима `Node.js -> Socket file`

### Что выставить в панели

1. Создай или открой сайт своего домена в `ispmanager`
2. В поле `Handler` выбери `Node.js`
3. Выбери подходящую версию Node.js, лучше LTS
4. В `Connection method` выбери:
   - `Port`, если хочешь самый простой вариант
   - `Socket file`, если хочешь запуск через unix-socket
5. Включи `Shell access` для пользователя сайта

### Развертка через Shell client

Открой `Sites -> нужный домен -> ... -> Shell client`, затем выполни:

Внимание: первая команда очищает текущую директорию сайта, кроме `.well-known`.

```bash
find . -mindepth 1 -maxdepth 1 ! -name '.well-known' -exec rm -rf {} +
git clone https://github.com/ObamaObama444/--- .
npm ci --omit=dev
```

После этого в панели:

1. Нажми `Npm install`, если хочешь, чтобы panel еще раз проверила зависимости
2. Нажми `Restart (Node.js)`

Если хочешь проверить процесс через shell:

```bash
/usr/lib/ispnodejs/bin/pm2 list
```

### Если нужен ручной запуск без панели

Можно поднять и через обычный `pm2`:

```bash
npm install -g pm2
npm ci --omit=dev
pm2 start ecosystem.config.js
pm2 save
```

## Структура данных

- сообщения: `data/messages.json`
- загруженные файлы: `uploads/`

Эти данные не должны храниться в Git. В репозитории лежат только `.gitkeep`.

## Ограничения текущей версии

- лимит одного файла: `25 МБ`
- хранится до `250` последних сообщений
- чат общий, без комнат и без приватных переписок
